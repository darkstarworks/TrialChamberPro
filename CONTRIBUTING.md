# Contributing to TrialChamberPro

Thank you for your interest in contributing to **TrialChamberPro**.

##  Important Notice

This project is maintained by a single developer. At this time:

* ⚠️ **Feature requests, enhancements, or design suggestions will only be accepted through discord**
* ❌ **Pull Requests will not be merged unless they are strictly bug fixes**
* ✅ **Bug reports are welcome and appreciated**

The goal is to keep the project stable and aligned with a specific vision.

---

## Reporting Bugs

If you encounter a bug, please report it using one of the following methods:

### Preferred: Discord

Join the Discord server and report your issue there:
https://discord.gg/aWMU2JNXex

### Alternative: GitHub Issues

Submit a bug report here:
https://github.com/darkstarworks/TrialChamberPro/issues

---

## Bug Report Guidelines

When reporting a bug, please include:

* A clear and descriptive title
* Steps to reproduce the issue
* Expected behavior
* Actual behavior
* Server version (e.g., Paper/Spigot + version)
* Plugin version
* Relevant logs or error messages (if available)

Incomplete reports may be ignored.

---

## Pull Requests

Pull Requests are **not encouraged**.

However, if you wish to submit one:

* It must fix a **specific, reproducible bug**
* It must not introduce new features, refactors, or stylistic changes
* It should be minimal and focused

PRs that do not meet these criteria will be closed.

---

## Final Notes

Your feedback is still valuable—even if not all suggestions are implemented.
If something is broken, reporting it helps improve the plugin for everyone.

Thanks for understanding.
